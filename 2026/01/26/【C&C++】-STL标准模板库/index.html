STL主要功能

容器 containers

  1. vector（动态数组）：

    空间连续、随机访问（ector也可以通过下标访问元素–>a[2]，当作数组遍历）。

    常用操作：

    操作类型 函数/写法 描述
    添加元素 v.push_back(10); 在尾部添加一个元素。vector基本上只提供尾插，除了尾插其他效率都非常低。
    访问元素 v[0] 像数组一样访问，不检查越界（快，但不安全）。
    v.at(0) 访问元素，如果越界会抛出异常（慢一点，但安全）。
    v.front() / v.back() 访问第一个 / 最后一个元素。
    删除元素 v.pop_back(); 删除最后一个元素。
    v.clear(); 清空所有元素。
    容量 v.size() 返回当前元素的个数。
    v.empty() 判断是否为空（返回 true/false）。

  2. map&unordered_map：

    是 C++ STL 中最常用的两个关联式容器。它们都用于存储 Key-Value (键-值对)，但在底层实现、性能和使用场景上有巨大的差异。

    特性 map unordered_map
    底层实现 红黑树 (Red-Black Tree) 平衡二叉搜索树 哈希表 (Hash Table)
    是否有序 有序 (默认按 Key 升序排序) 无序 (数据杂乱无章)
    查找速度 O(log n) (稳定) O(1) (平均)，最差 O(n)
    插入/删除 O(log n) O(1)(平均)
    内存占用 较低 (只存节点指针) 较高 (哈希表需要预留额外空间)
    适用场景 需要按顺序遍历 Key，或对查询稳定性要求极高。 只需要快速查找/插入，不在乎顺序。
    头文件 #include <map> #include <unordered_map>
    map

    红黑树；有序，默认按照key排序。

    map常用定义有：

    • map<int,int> m;

      m[5] = 10; m++;

    • map<string,bool> m;

      判断字符串是否被访问过：m['abc'] = true;

    操作类型 函数/写法 描述
    定义 map<数据类型1,数据类型2> m; 建立了一个从Key（整型）到Value（整型）的映射关系。
    添加修改 m[key] = val; 最常用的写法。如果 Key 不存在，则插入；如果 Key 已存在，则覆盖（更新）其 Value。
    访问元素 m[key] 返回 Key 对应的 Value。注意： 如果 Key 不存在，它会自动创建一个默认值（0或空串）插入 map 中。
    m.at(key) 返回 Key 对应的 Value。如果 Key 不存在，会抛出异常 (out_of_range)。比 [] 更安全，适合只读访问。
    查找/查询 m.find(key) 返回一个迭代器。如果找到，指向该元素；如果没找到，等于 m.end()。
    m.count(key) 判断 Key 是否存在。因为 map 中 Key 唯一，所以只会返回 1 (存在) 或 **0 (不存在)**。
    删除元素 m.erase(key); 删除指定 Key 的元素。返回被删除的个数（1 或 0）。
    m.clear(); 清空 map 中的所有元素。
    容量 m.size() 返回当前键值对的个数。
    m.empty() 判断容器是否为空（返回 true/false）。
    迭代器访问 it->first 访问当前迭代器指向元素的 **Key (键)**。
    it->second 访问当前迭代器指向元素的 **Value (值)**。

    map有一个强大的地方，可以对其元素直接进行运算。

    • 直接修改：因为 map[key] 返回的是 Value 的 **引用 (Reference)**，所以你对它的修改会直接改变 map 里的数据。

    • 自动初始化特性–>计数器：当对一个不存在的 Key 直接进行运算时，map 会先自动创建这个 Key，并将 Value 初始化为 0，然后再进行运算。
